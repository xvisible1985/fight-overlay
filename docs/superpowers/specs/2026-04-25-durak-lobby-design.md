## Обзор

Первый этап мини-игры «Дурак» в оверлее Nordheim. Охватывает виртуальный баланс монет, лобби столов и приглашение игроков. Игровой движок и UI карт — в Спеке 2.

---

## База данных (сервер nordheimunion.ru)

### `game_balances`
| Поле | Тип | Описание |
|---|---|---|
| `user_id` | FK → users | Привязка к жителю |
| `balance` | integer | Текущий баланс, default 1000 |
| `updated_at` | timestamp | Дата последнего изменения |

Запись создаётся автоматически при создании жителя (1 000 монет). Баланс постоянный — переходит между играми.

### `game_tables`
| Поле | Тип | Описание |
|---|---|---|
| `id` | uuid | Идентификатор стола |
| `name` | varchar | Название (задаёт создатель) |
| `type` | enum | `throwing` (подкидной) / `transfer` (переводной) |
| `max_players` | integer | 2–6 |
| `bet` | integer | Ставка в монетах (0 = без ставки) |
| `status` | enum | `waiting` / `playing` / `finished` |
| `created_by` | FK → users | Создатель |
| `created_at` | timestamp | |

### `game_table_players`
| Поле | Тип | Описание |
|---|---|---|
| `table_id` | FK → game_tables | |
| `user_id` | FK → users | |
| `seat_index` | integer | Порядок за столом (0-based) |
| `joined_at` | timestamp | |

## REST API (сервер)

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/api/games/balance` | Мой текущий баланс |
| `GET` | `/api/games/tables` | Список столов со статусом `waiting` |
| `POST` | `/api/games/tables` | Создать стол |
| `POST` | `/api/games/tables/:id/join` | Войти за стол |
| `DELETE` | `/api/games/tables/:id/leave` | Покинуть стол |
| `POST` | `/api/games/tables/:id/invite` | Пригласить по `game_name` |
| `POST` | `/api/games/tables/:id/start` | Начать игру (только создатель) |

### Валидация

- **Войти за стол:** `balance >= bet`, иначе `400 Недостаточно монет`
- **Старт игры:** повторная проверка баланса всех игроков стола; если у кого-то недостаточно — старт отменяется, стол остаётся в `waiting`, создатель получает ошибку «Игрок X не может оплатить ставку»; ставка списывается только при успешном старте
- **Создать стол:** `max_players` 2–6, `bet` ≥ 0

## WebSocket — лобби канал `/ws/games`

Клиент подключается при открытии виджета лобби. События (JSON):

```json
{ "type": "table_created",  "table": { ...tableFields } }
{ "type": "player_joined",  "table_id": "...", "player": { "game_name": "..." } }
{ "type": "player_left",    "table_id": "...", "user_id": "..." }
{ "type": "game_started",   "table_id": "...", "players": [ ... ] }
{ "type": "table_removed",  "table_id": "..." }
```

`game_started` получают только игроки за данным столом — это сигнал открыть игровое окно.

## Оверлей — виджет `wGames`

### Кнопка в Nordheim bar
Новая кнопка **🃏 Миниигры** — в сетке nb-btns на всю ширину (grid-column: 1 / -1), снизу.

### Виджет лобби
- Перетаскиваемый виджет в стиле существующих (`wChat`, `wFight`)
- Две вкладки: **Столы** (открытые, статус `waiting`) и **Мои игры** (столы где я участвую)
- Строка стола: название, тип (Подкидной / Переводной), `X/N игроков`, ставка, кнопка **Войти**
- Кнопка **Войти** недоступна если `balance < bet`
- Кнопка **＋ Создать стол** открывает форму
- Нижняя строка: 💰 Баланс: `N` монет + кнопка обновить
- Когда я за столом в статусе `waiting` и являюсь создателем — кнопка **▶ Начать игру** (активна когда ≥ 2 игроков)

### Форма создания стола
Поля:
1. **Вариант** — сегментный переключатель: Подкидной / Переводной
2. **Игроков** — сегментный переключатель: 2 / 3 / 4 / 5 / 6
3. **Ставка** — числовой input (0 = без ставки)
4. **Название** — текстовый input

### Правый клик на жителе → Пригласить
В контекстном меню жителя (`warriorCtx`) добавляется пункт **🃏 Пригласить за стол** — активен только если текущий пользователь сидит за столом со статусом `waiting`. Вызывает `POST /api/games/tables/:id/invite`.

Сервер отправляет приглашённому игроку WebSocket-событие:
```json
{ "type": "game_invite", "table_id": "...", "table_name": "...", "from": "game_name", "bet": 50 }
```
Оверлей показывает всплывающий попап (по аналогии с `#challenge`) с кнопками **Войти** / **Отказать**. Принятие = `POST /api/games/tables/:id/join`.

### Старт игры
При получении события `game_started` по WebSocket — оверлей открывает новое Electron-окно:  
`game-window.html?tableId=<id>&token=<token>&srv=<srv>`

## Виртуальный баланс

- **Начисление:** 1 000 монет при создании жителя (триггер на сервере)
- **Проверка при входе за стол:** `balance >= bet`
- **Проверка при старте:** повторная, у всех игроков
- **Списание:** в момент старта игры
- **Начисление выигрыша:** в Спеке 2
